# Funding Alignment

ClaimBound fits programs that support trustworthy AI, open digital commons,
reproducible research and public-interest data infrastructure.

The strongest proposal angle is not a single positive result. The strongest
angle is a reusable evidence discipline for public-source AI and ML claims.

## Best-Fit Themes

| Theme | Why ClaimBound fits |
| --- | --- |
| Trustworthy AI evaluation | ClaimBound fixes objectives, sources, gates and reporting before outcome inspection. |
| Open science infrastructure | Evidence cards make negative, blocked and reproduced outcomes easier to cite and inspect. |
| Public digital commons | The project is open-source, public-source oriented and designed for external review. |
| AI measurement practice | Protocols, result statuses and reproduction levels support repeatable evaluation. |
| Environmental and energy data | Early domains use official public sources with public value and clear baselines. |
| AI governance support | The project creates traceable records without becoming a certification authority. |
| Education and capacity building | Manual tracks teach reproducible ML discipline and honest reporting. |

## Weaker Fits

ClaimBound is a weaker fit for programs focused mainly on:

- production deployment;
- commercial product scaling;
- model training infrastructure;
- broad benchmark leaderboards;
- high-risk system certification;
- raw dataset hosting;
- blockchain or token ecosystems.

## Proposal Message

Use this message:

```text
ClaimBound builds open-source infrastructure for narrow, pre-registered,
public-source AI and ML evidence records. Each record preserves source lineage,
hashes, result status, claim boundary and reproduction level, including
negative and blocked outcomes.
```

Avoid this message:

```text
ClaimBound proves broad forecasting performance or model superiority.
```

## Suggested Work Packages

1. Evidence-card schema and validation.
2. Manual audit protocol and examples.
3. AI-assisted operator protocol and safeguards.
4. Public-source domain protocols.
5. Independent reproduction records.
6. Small registry index for evidence cards.
7. Documentation for external operators and reviewers.
